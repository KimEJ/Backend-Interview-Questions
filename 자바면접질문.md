# Java의 특징은 무엇인가요?

객체 지향 프로그래밍 (OOP) 언어입니다.
플랫폼 독립적인 언어로서, JVM(Java Virtual Machine) 위에서 실행됩니다.
자동 메모리 관리를 지원하는 가비지 컬렉션(Garbage Collection) 기능을 가지고 있습니다.
다양한 라이브러리와 프레임워크가 존재하여 개발 생산성이 높습니다.
멀티스레드를 지원합니다.

# Java에서 Garbage Collection이란 무엇인가요?

Garbage Collection은 자바에서 자동 메모리 관리를 위한 기능입니다.
Garbage Collector는 JVM에서 동작하며, 더 이상 사용되지 않는 객체를 자동으로 제거합니다.
이를 통해 개발자는 메모리 관리에 대한 부담을 덜 수 있으며, 더욱 안정적인 코드를 작성할 수 있습니다.

# Java에서 오버로딩과 오버라이딩의 차이점은 무엇인가요?

오버로딩(Overloading)은 같은 이름의 메소드를 다른 매개변수를 가지도록 여러 번 정의하는 것입니다.
오버라이딩(Overriding)은 부모 클래스의 메소드를 자식 클래스에서 재정의하는 것입니다.
즉, 오버로딩은 같은 이름의 다른 메소드를 만드는 것이고, 오버라이딩은 상속받은 메소드를 다시 정의하는 것입니다.

# Java에서 인터페이스와 추상 클래스의 차이점은 무엇인가요?

추상 클래스는 구현된 메소드와 추상 메소드가 모두 포함된 클래스입니다.
인터페이스는 모든 메소드가 추상 메소드인 특수한 형태의 추상 클래스입니다.
추상 클래스는 상속을 통해 자식 클래스에서 구현될 메소드를 일부 구현할 수 있지만, 인터페이스는 모든 메소드를 자식 클래스에서 구현해야 합니다.
또한, 자바에서는 다중 상속을 지원하지 않기 때문에, 인터페이스를 이용하여 여러 개의 클래스에서 동시에 구현될 수 있는 다중 상속을 구현할 수 있습니다.

# Java에서 예외 처리란 무엇인가요?

예외 처리는 프로그램 실행 중 발생할 수 있는 예기치 않은 상황을 대비하는 것입니다.
예외는 프로그램 실행 도중 발생한 문제로, 예를 들어 파일이 없는경우나 네트워크 연결이 끊긴 경우 등이 있습니다.

자바에서는 try-catch문을 사용하여 예외 처리를 합니다. try 블록 안에서 예외가 발생하면, catch 블록에서 해당 예외를 처리할 수 있습니다.
또한, throws 키워드를 사용하여 해당 메소드에서 발생 가능한 예외를 명시해줄 수 있습니다.

# Java에서 쓰레드(Thread)란 무엇인가요?

쓰레드는 하나의 프로세스 안에서 독립적으로 실행되는 실행 흐름입니다.
쓰레드를 이용하면 동시에 여러 가지 작업을 처리할 수 있습니다.
자바에서는 Thread 클래스를 상속받거나 Runnable 인터페이스를 구현하여 쓰레드를 만들 수 있습니다.

# Java에서 동기화(Synchronization)란 무엇인가요?

동기화는 멀티스레드 환경에서 공유 자원에 대한 접근을 제어하는 것입니다.
동기화를 하지 않으면 여러 쓰레드가 동시에 공유 자원에 접근하여 문제가 발생할 수 있습니다.
synchronized 키워드를 사용하여 동기화를 구현할 수 있습니다.

# Java에서 접근 제어자(Access Modifier)란 무엇인가요?

접근 제어자는 클래스, 메소드, 변수 등의 접근 범위를 지정하는 키워드입니다.
public은 어디서든 접근 가능한 공개 범위, protected는 같은 패키지나 자식 클래스에서만 접근 가능한 범위, default는 같은 패키지 내에서만 접근 가능한 범위, private은 같은 클래스 내에서만
접근 가능한 범위를 가지고 있습니다.

# Java에서 final 키워드는 어떤 역할을 하나요?

final 키워드는 변수, 메소드, 클래스 등을 선언할 때 사용됩니다.
변수에 final을 붙이면, 해당 변수는 상수가 되어 값을 변경할 수 없습니다.
메소드에 final을 붙이면, 해당 메소드를 자식 클래스에서 오버라이딩할 수 없습니다.
클래스에 final을 붙이면, 해당 클래스를 상속할 수 없습니다.

# Java에서 자바빈(Java Bean)이란 무엇인가요?

자바빈은 데이터를 표현하는 자바 클래스를 의미합니다.
일반적으로 데이터를 저장하기 위한 멤버 변수와 그 변수들에 접근하기 위한 Getter, Setter 메소드를 가지고 있습니다.
자바빈은 다른 자바 클래스에서 사용하기 위한 데이터를 담는 용도로 많이 활용됩니다. 예를 들어, 데이터베이스에서 조회한 데이터를 자바빈 객체에 담아서 전달하거나, 웹 어플리케이션에서 사용자가 입력한 데이터를 자바빈
객체에 담아서 처리하는 등의 용도로 사용됩니다.

# Java에서 제네릭(Generic)이란 무엇인가요?

제네릭은 클래스 또는 메소드에서 사용하는 데이터 타입을 컴파일 시에 결정하는 방식입니다.
제네릭을 사용하면 타입 안정성을 보장할 수 있으며, 코드의 재사용성과 가독성을 높일 수 있습니다.
예를 들어, ArrayList 클래스를 선언할 때, ArrayList<String>과 같이 제네릭을 이용하여 String 타입의 데이터만 담을 수 있도록 할 수 있습니다.

# Java에서 람다식(Lambda Expression)이란 무엇인가요?

람다식은 함수형 프로그래밍에서 사용되는 개념으로, 간단한 함수를 만들기 위한 식입니다.
람다식은 메소드를 하나의 식으로 표현한 것으로, 매개변수와 반환값이 있을 수 있습니다.
자바 8부터 람다식이 도입되었으며, 함수형 인터페이스와 함께 많이 사용됩니다.

# Java에서 열거형(Enum)이란 무엇인가요?

열거형은 상수를 표현하는 자바의 특별한 데이터 타입입니다.
열거형은 일반적으로 관련된 상수들을 그룹화하여 사용할 때 유용합니다.
열거형은 자바에서 enum 키워드를 사용하여 선언합니다.

# Java에서 데코레이터(Decorator) 패턴이란 무엇인가요?

데코레이터 패턴은 객체지향 디자인 패턴 중 하나입니다.
데코레이터 패턴은 기존 객체에 새로운 기능을 추가하는 방식으로 확장성을 높이는 패턴입니다.
예를 들어, 자바에서는 InputStream 클래스에 대한 데코레이터로 BufferedInputStream, DataInputStream 등이 있습니다.

# Java에서 직렬화(Serialization)란 무엇인가요?

직렬화는 자바 객체를 바이트 형태로 변환하는 과정을 의미합니다.
직렬화를 통해 객체를 파일에 저장하거나, 네트워크를 통해 전송할 수 있습니다.
자바에서는 Serializable 인터페이스를 구현하여 직렬화를 할 수 있습니다.

# Java에서 자료구조와 알고리즘을 사용하는 이유는 무엇인가요?

자료구조와 알고리즘은 프로그램의 성능과 효율성에 큰영향을 미치기 때문에 자바 개발자는 이를 잘 이해하고 적절히 활용해야 합니다.

자료구조는 데이터를 구조화하여 저장하고, 검색하거나 조작하기 위한 방법을 제공합니다. 즉, 데이터의 저장과 접근을 효율적으로 하기 위해 필요합니다.
알고리즘은 문제를 해결하기 위한 단계적인 절차를 의미합니다. 알고리즘을 사용하면 문제를 더욱 효율적으로 해결할 수 있습니다.
따라서, 자바 개발자는 다양한 자료구조와 알고리즘을 이해하고, 적절하게 활용하여 높은 성능과 효율성을 가진 프로그램을 개발할 수 있어야 합니다.

# Java에서 자바 스크립트(JavaScript)와의 차이점은 무엇인가요?

자바 스크립트는 웹 브라우저에서 실행되는 클라이언트 사이드 스크립트 언어입니다.
자바는 서버 사이드 언어이며, 다양한 플랫폼에서 실행될 수 있습니다.
또한, 자바는 객체 지향 언어이며, 자바 스크립트는 객체 기반 언어입니다.
자바는 컴파일러를 사용하여 실행 파일을 생성하고, 자바 스크립트는 인터프리터를 사용하여 실행됩니다.

# Java에서 JDBC란 무엇인가요?

JDBC(Java Database Connectivity)는 자바에서 데이터베이스에 접속하기 위한 API입니다.
JDBC를 사용하면 자바에서 다양한 데이터베이스에 접속하여 데이터를 조회, 수정, 삭제할 수 있습니다.
JDBC는 데이터베이스 벤더에서 제공하는 드라이버를 사용하여 데이터베이스와 연결하며, 자바 애플리케이션과 데이터베이스 간의 통신을 담당합니다.

# Java에서 컬렉션(Collection)이란 무엇인가요?

컬렉션은 자바에서 데이터를 저장하는 방법 중 하나입니다.
컬렉션을 사용하면 데이터를 쉽게 관리할 수 있으며, 검색, 추가, 삭제 등의 연산을 효율적으로 수행할 수 있습니다.
자바에서는 다양한 컬렉션 클래스를 제공하며, List, Set, Map 등이 있습니다.

# Java에서 스프링(Spring) 프레임워크란 무엇인가요?

스프링 프레임워크는 자바 기반의 애플리케이션을 개발하기 위한 오픈 소스 프레임워크입니다.

스프링은 객체 지향 프로그래밍과 의존성 주입(Dependency Injection), 제어 역전(Inversion of Control) 등의 개념을 활용하여 개발 생산성과 유지 보수성을 높일 수 있습니다.
스프링은 다양한 모듈로 구성되어 있으며, 대표적으로 스프링 MVC, 스프링 데이터, 스프링 시큐리티 등이 있습니다.
스프링은 DI, AOP 등의 개념을 기반으로 하여 다양한 어플리케이션을 만들 수 있습니다.

# Java에서 RESTful API란 무엇인가요?

RESTful API는 Representational State Transfer(표현 상태 전이)의 약자입니다.
RESTful API는 자원을 URL로 표현하고, HTTP 메소드를 이용하여 해당 자원을 처리하는 방식으로 동작합니다.
RESTful API를 사용하면 다양한 클라이언트가 서버의 자원에 접근하여 데이터를 주고받을 수 있습니다.
Java에서는 스프링 프레임워크에서 RESTful API를 구현하는 데에 많이 활용됩니다.

# Java에서 ORM(Object Relational Mapping)이란 무엇인가요?

ORM은 객체 지향 프로그래밍 언어와 관계형 데이터베이스의 데이터를 매핑하는 기술입니다.
ORM을 사용하면 자바 객체와 데이터베이스의 테이블 간의 변환 작업을 자동화하여 생산성을 높일 수 있습니다.
대표적인 Java ORM 프레임워크로는 Hibernate, MyBatis 등이 있습니다.

# Java에서 스트림(Stream)이란 무엇인가요?

스트림은 자바 8부터 도입된 함수형 프로그래밍 방식의 데이터 처리 기능입니다.
스트림을 이용하면 컬렉션과 같은 데이터를 필터링, 매핑, 정렬하는 등의 작업을 쉽게 수행할 수 있습니다.
스트림은 중간 연산과 최종 연산으로 구성되며, 중간 연산은 다른 스트림을 반환하고 최종 연산은 스트림을 반환하지 않습니다.

# Java에서 함수형 인터페이스(Functional Interface)란 무엇인가요?

함수형 인터페이스는 단 하나의 추상 메소드만을 가지는 인터페이스입니다.
함수형 인터페이스를 이용하여 람다식을 구현할 수 있으며, 자바에서는 java.util.function 패키지에서 다양한 함수형 인터페이스를 제공합니다.

# Java에서 JPA(Java Persistence API)란 무엇인가요?

JPA는 자바에서 ORM을 구현하기 위한 API입니다.
JPA는 애플리케이션과 데이터베이스 간의 객체 매핑을 쉽게 처리할 수 있도록 돕습니다.
JPA는 자바에서 ORM을 구현하기 위한 대표적인 표준 기술입니다.

# Java에서 멀티스레드(Multi-thread)란 무엇인가요?

멀티스레드는 하나의 프로세스에서 여러 개의 스레드를 실행하는 방식입니다.
자바에서 멀티스레드를 구현하려면 Thread 클래스를 상속받거나 Runnable 인터페이스를 구현해야 합니다.
멀티스레드를 사용하면 여러 작업을 동시에 처리할 수 있어 프로그램의 성능을 높일 수 있습니다.

# Java에서 가비지 컬렉션(Garbage Collection)이란 무엇인가요?

가비지 컬렉션은 자바에서 자동으로 메모리를 관리하기 위한 기능입니다.
가비지 컬렉션은 더 이상 사용하지 않는 객체를 자동으로 제거하여 메모리 누수를 방지합니다.
자바에서는 JVM(Java Virtual Machine)이 가비지 컬렉션을 수행하며, 이를 조정하기 위한 다양한 옵션들이 제공됩니다.

# Java에서 예외(Exception)와 오류(Error)의 차이점은 무엇인가요?

예외는 프로그램에서 예상할 수 있는 상황에서 발생하는 오류를 말합니다.
오류는 프로그램에서 예상하지 못한 상황에서 발생하는 오류를 말합니다.
예외는 try-catch 블록을 이용하여 예외 처리를 할 수 있으며, 오류는 프로그램을 강제 종료시키는 것이 일반적입니다.

# Java에서 인터페이스(Interface)란 무엇인가요?

인터페이스는 자바에서 클래스와 클래스 사이의 커뮤니케이션을 위한 명세서 역할을 하는 일종의 추상 클래스입니다.
인터페이스는 추상 메소드와 상수 필드를 정의할 수 있으며, 클래스에서 인터페이스를 구현할 경우 인터페이스에서 정의한 메소드를 반드시 구현해야 합니다.
자바에서 인터페이스는 다중 상속을 지원합니다.

# Java에서 스태틱(Static) 키워드란 무엇인가요?

스태틱 키워드는 클래스 변수와 클래스 메소드를 정의할 때 사용됩니다.
스태틱 변수는 객체가 생성되지 않아도 클래스로부터 직접 접근할 수 있는 변수입니다.
스태틱 메소드는 객체가 생성되지 않아도 클래스로부터 직접 호출할 수 있는 메소드입니다.
